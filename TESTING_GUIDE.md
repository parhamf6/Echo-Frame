# Echo Frame HLS Processing & Upload Testing Guide

Complete step-by-step guide to process videos into HLS format and upload them to the backend.

---

## Prerequisites

### 1. Install FFmpeg

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install ffmpeg
```

**macOS:**
```bash
brew install ffmpeg
```

**Windows:**
- Download from https://ffmpeg.org/download.html
- Extract and add to `PATH`

**Verify installation:**
```bash
ffmpeg -version
ffprobe -version
```

### 2. Python 3.10+

```bash
python3 --version  # Should be 3.10 or higher
```

### 3. PostgreSQL Database

Ensure PostgreSQL is running and you have connection credentials.

## Step 1: Backend Setup

### 1.1 Navigate to Backend Directory

```bash
cd /home/parhamf/codes/Echo-Frame/backend/echoframe_api
```

### 1.2 Create Environment File

Create `.env` file in `backend/echoframe_api/`:

```bash
cat > .env << 'EOF'
# Database
DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/echoframe
DATABASE_URL_SYNC=postgresql://user:password@localhost:5432/echoframe

# Redis
REDIS_URL=redis://localhost:6379/0

# Security
SECRET_KEY=your-secret-key-change-this-in-production-min-32-chars
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
REFRESH_TOKEN_EXPIRE_DAYS=7

# Application
APP_NAME=EchoFrame
DEBUG=True
ENVIRONMENT=development

# CORS
ALLOWED_ORIGINS=["http://localhost:3000","http://localhost:8000"]

# Rate Limiting
RATE_LIMIT_PER_MINUTE=60
GUEST_RATE_LIMIT_PER_HOUR=3
GUEST_RATE_PERIOD_SECONDS=3600
RATE_LIMIT_KEY_PREFIX=rl

# File Upload
MAX_UPLOAD_SIZE=10737418240
ALLOWED_VIDEO_EXTENSIONS=[".m3u8",".ts",".mp4",".mkv"]
ALLOWED_SUBTITLE_EXTENSIONS=[".vtt",".srt",".ass"]

# Storage paths (relative to backend/echoframe_api/)
STORAGE_PATH=storage
VIDEO_STORAGE_PATH=storage/videos
SUBTITLE_STORAGE_PATH=storage/subtitles
EOF
```

**⚠️ Important:** Replace `user`, `password`, and database name with your actual PostgreSQL credentials.

### 1.3 Create Virtual Environment (if not already created)

```bash
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

### 1.4 Install Dependencies

```bash
pip install -r requirements.txt
```

### 1.5 Run Database Migrations

```bash
alembic upgrade head
```

### 1.6 Create Admin User (if needed)

```bash
python scripts/create_admin.py
# Follow prompts to create admin username/password
```

### 1.7 Start Backend Server

```bash
uvicorn app.main:socket_app --reload --host 0.0.0.0 --port 8000
```

**Expected output:**
```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

**Keep this terminal open!** The server must be running for uploads to work.

## Step 2: CLI Setup

### 2.1 Navigate to CLI Directory

Open a **new terminal** (keep backend running):

```bash
cd /home/parhamf/codes/Echo-Frame/cli
```

### 2.2 Create Virtual Environment (if not already created)

```bash
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

### 2.3 Install CLI in Editable Mode

```bash
pip install -e .
```

### 2.4 Verify CLI Installation

```bash
echo-frame --help
```

You should see:
```
Usage: echo-frame [OPTIONS] COMMAND [ARGS]...

  Echo Frame CLI utilities

Options:
  --help  Show this message and exit.

Commands:
  config   Configuration management
  process  Process video inputs into HLS packages
  upload   Upload processed assets to Echo Frame backend
```

### 2.5 Configure CLI

#### 2.5.1 Initialize Configuration

```bash
echo-frame config init
```

**Prompts:**
- `server_url`: `http://localhost:8000` (or your backend URL)
- `admin_token`: Leave blank for now (we'll get it in Step 3)
- `ffmpeg_path`: Leave blank to auto-detect

#### 2.5.2 Get Admin Token

In another terminal, login to get your admin token:

```bash
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "your_admin_username", "password": "your_admin_password"}'
```

**Response:**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "expires_in": 1800
}
```

Copy the `access_token` value.

#### 2.5.3 Set Admin Token in CLI Config

```bash
echo-frame config set admin_token eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

#### 2.5.4 Verify Configuration

```bash
echo-frame config show
```

Should display:
```
Server URL: http://localhost:8000
Token: eyJh***...***xyz
FFmpeg Path: (auto-detected)
```

## Step 3: Process a Video into HLS

### 3.1 Prepare a Test Video

Find a video file to process (e.g., `~/videos/test.mp4`). For testing, a short clip (30-60 seconds) works best.

### 3.2 Process Video (Transcode Mode)

This will create multiple quality variants:

```bash
echo-frame process run \
  --source ~/videos/test.mp4 \
  --targets "1080,720,480,360" \
  --output-dir ./processed_video
```

**What happens:**
- FFmpeg transcodes the video into 4 quality tiers
- Creates HLS segments (`.ts` files) for each quality
- Generates `master.m3u8` and variant playlists (`1080/index.m3u8`, etc.)
- Captures thumbnail (`thumbnails/poster.jpg`)
- Creates `metadata.json` with video info

**Expected output:**
```
Processing video: test.mp4
Creating HLS ladder: 1080, 720, 480, 360
...
✓ Processing complete!
Output directory: ./processed_video
```

### 3.3 Verify Output Structure

```bash
ls -la processed_video/
```

Should show:
```
master.m3u8
1080/
   index.m3u8
   segment_00001.ts
   segment_00002.ts
   ...
720/
480/
360/
thumbnails/
  poster.jpg
metadata.json
logs/
```

### 3.4 (Optional) Process with Subtitles

If you have subtitle files:

```bash
echo-frame process run \
  --source ~/videos/test.mp4 \
  --targets "1080,720,480" \
  --subtitle captions_en.srt --lang en \
  --subtitle captions_es.srt --lang es \
  --output-dir ./processed_video
```

This will create `subtitles/en.vtt` and `subtitles/es.vtt` in the output directory.

## Step 6: Configure Nginx for Video Serving (Optional)

For production, configure Nginx to serve the `/videos/` directory with CORS headers.

### 6.1 Create Nginx Configuration

Create `/etc/nginx/sites-available/echoframe`:

**For local testing, use `localhost` or add to `/etc/hosts` (see below):**

```nginx
server {
    listen 80;
    # For local testing, use localhost or your custom domain
    # Option 1: Use localhost (simplest)
    server_name localhost;
    
    # Option 2: Use custom domain (add to /etc/hosts first)
    # server_name echoframe-nginx.com;

    # Serve video files with CORS
    location /videos/ {
        alias /home/parhamf/codes/Echo-Frame/backend/echoframe_api/storage/videos/;
        
        # CORS headers for HLS streaming
        add_header Access-Control-Allow-Origin * always;
        add_header Access-Control-Allow-Headers * always;
        add_header Access-Control-Allow-Methods "GET, OPTIONS" always;
        add_header Access-Control-Expose-Headers "Content-Length,Content-Range" always;

        # Handle CORS preflight (OPTIONS requests)
        if ($request_method = OPTIONS) {
            add_header Access-Control-Allow-Origin * always;
            add_header Access-Control-Allow-Headers * always;
            add_header Access-Control-Allow-Methods "GET, OPTIONS" always;
            add_header Content-Length 0;
            add_header Content-Type text/plain;
            return 204;
        }

        # HLS MIME types
        types {
            application/vnd.apple.mpegurl m3u8;
            video/mp2t ts;
            image/jpeg jpg;
            text/vtt vtt;
        }
        default_type application/octet-stream;

        # Caching
        expires 1h;
    }

    # Proxy API requests to FastAPI
    location /api/ {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

**For local testing with a custom domain:**

If you want to use `echoframe-nginx.com` locally, add it to your `/etc/hosts` file:

```bash

```

Add this line:
```

```

Then update the `server_name` in the Nginx config to `echoframe-nginx.com` and reload Nginx.

## Troubleshooting

### Upload Fails with 401

- Check admin token: `echo-frame config show`
- Token may have expired; get a new one: `curl -X POST http://localhost:8000/api/v1/auth/login ...`

### Upload Fails with 400 "master.m3u8 not found"

- Verify the processed directory structure: `ls processed_video/master.m3u8`
- Ensure you're uploading the directory that contains `master.m3u8`, not a parent directory

### Backend Server Not Starting

- Check PostgreSQL is running: `sudo systemctl status postgresql`
- Check Redis is running: `redis-cli ping` (should return `PONG`)
- Verify `.env` file exists and has correct database credentials

### CLI Not Found

- Ensure virtual environment is activated: `source venv/bin/activate`
- Reinstall CLI: `pip install -e .`

### Files Not Accessible via Nginx

- Check file permissions: `ls -la storage/videos/`
- Verify Nginx `alias` path matches actual storage location
- Check Nginx error logs: `sudo tail -f /var/log/nginx/error.log`
